## Component Details

### 1. NPU DMA Fix (Kernel Patch)

The Qualcomm fastrpc driver (which mediates communication with the Hexagon DSP) requires a reserved DMA memory pool in the Device Tree. Stock kernel 6.18.x ships without this pool, causing:

```
no reserved DMA memory for FASTRPC
Error 14001 (AEE_EUNSUPPORTED)
```

**Fix**: Patch the Device Tree Blob (DTB) on the EFI System Partition to add:
- A `shared-dma-pool` at `0x89000000` (32 MB)
- `memory-region` property on both CDSP (`remoteproc@a300000`) and ADSP (`remoteproc@3700000`) fastrpc nodes

See `docs/02-npu-dma-fix.md` for the full procedure.

### 2. Model

- **Models**:
  - **Llama 3.2 1B** вҖ” QNN HTP context binary, INT8, 4096 context
  - **Qwen2.5-Coder-0.5B** вҖ” QNN HTP context binary, INT8, 32768 context
  - **Qwen2.5-Coder-1.5B** вҖ” QNN HTP context binary, INT8, 32768 context
- **Format**: QNN HTP context binary (`.serialized.bin`) вҖ” pre-compiled for Hexagon v68
- **Source**:
  - Llama: [ModelScope вҖ” radxa/Llama3.2-1B-1024-qairt-v68](https://modelscope.cn/models/radxa/Llama3.2-1B-1024-qairt-v68) (recompiled for 4096 context)
  - Qwen: Custom build via `qnn-context-binary-generator` pipeline
- **Size**:
  - Llama: 1.7 GB (INT8 quantized, 4096 context window)
  - Qwen 0.5B: 310 MB (INT8 quantized, 32768 context window)
  - Qwen 1.5B: ~2 GB (INT8 quantized, 32768 context window)

### 3. genie-rs API Server

The API server is built in Rust using:
- **axum** вҖ” HTTP framework (OpenAI-compatible routes)
- **Genie SDK C bindings** вҖ” generated by bindgen from QAIRT headers
  - `GenieDialogConfig_createFromJson()` вҖ” loads HTP config
  - `GenieDialog_create()` вҖ” creates inference context
  - `GenieDialog_query()` вҖ” runs inference with streaming callback
- **HTP backend** вҖ” `QnnHtp` type with `QnnHtpV68Stub`/`Skel`
- **`--working-dir`** вҖ” enables running from the model directory (required for Skel loading)

### 4. Boot Process

The Dragon Q6A boots via **UEFI + systemd-boot** (not U-Boot):

```
UEFI вҶ’ systemd-boot вҶ’ kernel + initrd + DTB from ESP
```

- DTB: `/boot/efi/RadxaOS/<kernel-version>/qcs6490-radxa-dragon-q6a.dtb`
- Kernel: `/boot/efi/RadxaOS/<kernel-version>/linux`
- Boot entry: `/boot/efi/loader/entries/RadxaOS-<kernel-version>.conf`

### 5. Memory Map

```
0x82700000  cdsp-secure-heap@82700000    (CDSP secure memory)
0x82710000  в”Җв”Җ end of cdsp-secure-heap
    ...
0x89000000  fastrpc@89000000  (32 MB)    вҶҗ NEW: shared-dma-pool for fastrpc
0x8b000000  в”Җв”Җ end of pool
    ...
0x8b800000  adsp@8b800000                 (ADSP region)
```

The 32 MB pool sits in a ~144 MB gap between CDSP and ADSP regions.

## Key Constraints
| **Context window** | 4096 (Llama), 32768 (Qwen 0.5B/1.5B) |
| **Memory** | ~3.5 GB during inference (1.7 GB Llama) to ~4 GB (1.5B Qwen) |
| **NPU only** | Uses QNN HTP (Hexagon v68). GenAiTransformer CPU fallback available |
| **Tool calling** | Server-side routing вҖ” not model-dependent |
